## Stripe / commerce mechanics

### How the checkout flow works

1. Customer fills booking form on `bronsbeach.com`
2. `BookingForm.tsx` POSTs to `/api/checkout` with `{name, phone, email, product, pickupDate, returnDate, accessPoint, numDays}`
3. Server validates fields, looks up product in catalog (server-trusted), computes `totalCents`, splits into `platformFeeCents` (12%) + `vendorCents` (88%)
4. Server creates Stripe Checkout Session with metadata embedding the split
5. If `STRIPE_BRONS_CONNECTED_ACCT_ID` is set → Direct Charge mode (88% auto-routes to connected account); if unset → charge lands in platform account, split is metadata-only
6. Browser redirected to Stripe-hosted checkout page
7. Customer pays → redirect to `/checkout/success?session_id=...`
8. Success page renders confirmation copy (no email fired yet — that's a pending build item)

### Date UX (category-aware)

- **Beach products**: single "Setup date" field by default. Most beach rentals are single-day. Optional `+ Need a setup for multiple days?` link reveals end-date field.
- **Cart products**: range "Pickup" + "Return" — always multi-day-shaped.

When category switches (user picks a cart after picking beach), `multiDay` toggles automatically.

### Server-side trust pattern

Per the price-IDOR fix pattern from PAL: the API NEVER trusts client-supplied prices or fee splits. Server reads catalog (`getProduct(slug)`), computes `totalCents` and `splitTotal()` from trusted source, ignores any client-provided values. A manipulated request body can't underpay or shift the fee split.

## Auth model

`/admin/*` gated by HTTP basic auth via `src/middleware.ts`:
- Username: `admin` (hard-coded)
- Password: `process.env.ADMIN_PASSWORD`
- Returns 503 if `ADMIN_PASSWORD` not set (won't expose admin surfaces with no protection)
- Browser native auth dialog; no login form to maintain

When `ADMIN_PASSWORD` env var is set in Vercel + redeploy completes:
1. Visit `bronsbeach.com/admin`
2. Browser pops auth dialog
3. Enter `admin` / `<password>`
4. Browser remembers; visit other `/admin/*` paths without re-auth

## Build / deploy mechanics

**To deploy:**
```
git push origin main
# Vercel auto-deploys ~60 sec
```

**To trigger Vercel rebuild without code changes:**
```
git commit --allow-empty -m "trigger redeploy"
git push
```

**To check what's in Vercel runtime env:**
```
curl https://bronsbeach.com/api/diagnose | jq
```

**To test checkout API directly:**
```
curl -X POST https://bronsbeach.com/api/checkout \
  -H 'Content-Type: application/json' \
  -d '{"name":"Test","phone":"5125551234","email":"test@test.com","product":"chair-umbrella","pickupDate":"2026-05-15","returnDate":"2026-05-15","accessPoint":"Access Rd 1A","numDays":1}'
# Returns {url: "https://checkout.stripe.com/..."} on success
```
