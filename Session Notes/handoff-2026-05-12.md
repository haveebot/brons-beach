## Vercel infrastructure rebuild (locked 2026-05-11 PM-3)

Hit a sequence of Vercel auth walls onboarding Collie. Each one taught a systemic gap. Final answer: **make tenant repos PUBLIC on GitHub.** Full diagnosis filed at workspace memory `feedback_vercel_contributor_onboarding.md`.

### What we tried and what actually worked

| Setup | API trigger | Git-push trigger |
|---|---|---|
| Contributor team role = MEMBER on private repo | ❌ blocked | ❌ blocked |
| Contributor team role = DEVELOPER on private repo | ✅ works | ❌ **still blocked** |
| Contributor team role = MEMBER + project-level `PROJECT_DEVELOPER` | API rejects: `Invalid role combination` | N/A |
| Contributor team role = DEVELOPER + project-level `PROJECT_DEVELOPER` | API rejects: `Invalid role combination` | N/A |
| **Public repo + any team role** | **✅ works** | **✅ works** |

**Pro plan limitation:** no granular project-level membership for non-OWNERs. The only sustainable fix is the public-repo flip.
